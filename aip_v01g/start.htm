Artificial Intelligence Program (AIP) Version 0.1

AIP is a program that has the capacity and ability to learn to play simple games board games such as Tic Tac Toe. However the program has no knowledge of the game nor what constitutes a win.

AIP must be taught by the user (teacher) to play the game. AIP is only capable of selecting random moves the very first time it plays. As the teacher wins a game, he or she highlights the pattern that constitutes the win. This winning pattern is sent to the data array and is available for processing by AIP.

AIP has an intricate pattern recognition ability. It can take a pattern from a 3 x 3 grid and manipulate it several ways in an effort to increase its knowledge base. Each win by the teacher or program reinforces certain acquired knowledge.

After playing a handful of games such as Tic Tac Toe, AIP will learn the game and will soon be able to block winning plays by the teacher and will strive to win itself. Eventually, the teacher will find the program unbeatable (theoretical).

Allowable Game Boards

AIP v0.1 recognizes three different game board sizes. The boards are organized into grids. They are square grids meaning the number of rows is equal to the number of columns.

     3x3 Grid                         4x4 Grid                           5x5 Grid

The 3x3 grid has 9 playable cells, the 4x4 has 16 playable cells and the 5x5 has 25 playable cells.

AIP v0.1 is limited to Game Type =1.  This describes a game that uses a grid-like board that is squared (rows = columns).  Additionally, neither the program nor the teacher have game pieces on the board. It is empty.  The teacher select who moves first, teacher or the program.  A game piece is identified by an alphanumeric character. For example: 'X' and 'O'.  The goal of a Type 1 game is to move the pieces to conform to a particular pattern.  These patterns must be taught by the teacher during the course of playing several games against the computer.

Winning Patterns

The interesting aspect of AIP is that there are no game rules. The teacher is free to design and develop their own games with its own unique winning patterns.  In fact, AIP can handle games that allow for more than one pattern of its pieces to constitute a win.  One limitation  is a win can only consist of the same pieces, all 'X's or all 'O's but not mixed.

Bit Representation of Winning Pattern

A 32-bit number is used to hold a winning pattern.  The 32-bit number is also known as a long integer.  Consider the 3x3 game board below. It contains only 9 playable cells.  Therefore, it uses only the lower 9 bits of a 32 bit number.

Each cell above is identified by a unique b? value. The 'b' stands for 'bit'.  'b0' is the least significant bit. A winning pattern may be defined by setting the appropriate bit to a value of '1' if that cell is part of a winning pattern.   A '0' is placed in the binary number if the cell was not part of the winning pattern.  What bits would be set (equal to 1) for the below win?

The correct answer would be b0, b4 and b8.  Expressed in a 32 bit binary number, it would look like: 00000000000000000000000100010001

Only part of the 32-bit number is shown above. The nine digit binary number is divided into three parts, each with three bits. The least significant bit is 'b0' which corresponds to the top-left cell in the 3x3 grid.  Reading from RIGHT to LEFT, the first three bits correspond to the top row or row= 1. The center three bits correspond to the middle row and the last three bits represent the bottom row.
